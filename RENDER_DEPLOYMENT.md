# Развёртывание на Render.com

Полная инструкция по развёртыванию социальной сети на Render.com.

## Предварительные условия

- GitHub аккаунт (готов)
- Аккаунт на [Render.com](https://render.com)
- Репозиторий загружен на GitHub (готов)

## Пошаговая инструкция

### 1. Создайте аккаунт на Render.com

1. Перейдите на https://render.com
2. Нажмите **Sign up** → выберите **GitHub**
3. Авторизуйтесь с вашим GitHub аккаунтом
4. Дайте разрешение Render на доступ к репозиториям

### 2. Создайте новый Web Service

1. В панели Render нажмите **+ New**
2. Выберите **Web Service**
3. В поле поиска выберите репозиторий: `g26sa/social_main`
4. Нажмите **Connect**

### 3. Конфигурируйте Web Service

Заполните поля:

| Поле | Значение | Описание |
|------|----------|---------|
| **Name** | `social-network` | Название приложения (будет в URL) |
| **Region** | Выберите ближайший | Например `Frankfurt (eu-west-1)` |
| **Branch** | `main` | Ветка для развёртывания |
| **Runtime** | `PHP 8` | Язык и версия |
| **Build Command** | *(оставить пусто)* | PHP не требует сборки |
| **Start Command** | `php -S 0.0.0.0:$PORT` | Запуск встроенного веб-сервера |

### 4. Добавьте переменные окружения

В секции **Environment Variables** добавьте:

```
DB_HOST=<postgresql-host>  # Предоставленный хост БД
DB_USER=<username>          # Ваш пользователь БД
DB_PASSWORD=<password>      # Ваш пароль БД
DB_NAME=social              # Имя базы данных
APP_ENV=production
APP_DEBUG=false
```

#### Совет: Используйте встроенную PostgreSQL на Render

1. Нажмите **Create a database** (если предложит)
2. Render создаст PostgreSQL БД
3. Автоматически заполнит переменные окружения

### 5. Настройте план (бесплатный)

- Выберите **Free plan**
- Ограничения: 0.5 GB RAM, спящий режим при неактивности
- Идеально для демонстрации проекта

### 6. Развёртывание

1. Нажмите **Create Web Service**
2. Render начнёт развёртывание (можно смотреть логи)
3. Разворачивание занимает 2-5 минут

### 7. Получите URL вашего приложения

После успешного развёртывания вы получите URL вроде:
```
https://social-network.onrender.com
```

Это URL для кнопки на вашем статическом сайте! 🎉

## ⚠️ Важные замечания

### Для первого запуска

Если вы используете PostgreSQL (рекомендуется на Render), нужно:

1. **Адаптировать код для PostgreSQL** или использовать MySQL

   Render лучше всего работает с PostgreSQL. Если вам нужен MySQL:
   - Используйте ClearDB MySQL: дополнительная услуга
   - Или используйте другой хостинг (Railway.app лучше для MySQL)

2. **Инициализировать БД**

   Через SSH консоль Render или через phpmyadmin:
   ```bash
   mysql -h $DB_HOST -u $DB_USER -p$DB_PASSWORD $DB_NAME < schema.sql
   ```

### Холодные запуски

На бесплатном плане Render переводит неактивные приложения в спящий режим. Первый запрос может занять 30-50 секунд.

### Размер файлов

Максимум 100 MB на бесплатном плане. Если ваши uploads слишком большие, нужно переместить их на AWS S3 или другое хранилище.

## 🚨 Проблемы и решения

### "Build failed"
- Проверьте что все `.php` файлы синтаксически корректны
- Смотрите логи в панели Render

### "Domain not found" или "502 Bad Gateway"
- Дождитесь 2-5 минут развёртывания
- Обновите страницу (Cmd+Shift+R)
- Проверьте логи в Render панели

### Ошибка подключения к БД
- Проверьте переменные окружения совпадают с `.env.example`
- Убедитесь что БД создана и инициализирована

### Файлы uploads не сохраняются
- На Render файлы хранятся временно и удаляются при пересоздании контейнера
- Решение: настроить S3 или другое облачное хранилище

   ```bash
   git push origin main
   ```
